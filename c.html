Tideman

Esse código em C é um programa que implementa o algoritmo de votação do método de Tideman, que é um sistema de votação que tenta escolher um vencedor levando em consideração as preferências dos eleitores em relação aos candidatos. Aqui está uma explicação passo a passo do código:

  1. Inclusão de bibliotecas:
    O programa inclui algumas bibliotecas padrão do C, como cs50.h e stdio.h, necessárias para funções de entrada/saída e manipulação de strings.
  2. Definições e Constantes:
    O programa introduz constantes que incluem o valor máximo permitido para candidatos (MAX), um número de preenchimento (FILLER_NUM) usado para preencher lacunas, e uma flag booleana (HAS_NO_EDGE) para indicar a ausência de uma aresta. Também inclui a definição da estrutura 'pair' para armazenar o vencedor e o perdedor de uma comparação de votos.
  3. Variáveis Globais:
    O programa declara variáveis globais, incluindo matrizes usadas para armazenar preferências e bloqueios, juntamente com arrays para armazenar candidatos e pares.
  4. Funções Principais:
    main(): Essa função serve como a peça central do programa. Ela trata da entrada do usuário, coleta os votos, calcula as preferências, cria pares, os classifica, aplica bloqueios e, por fim, imprime o vencedor da eleição.
  5. Funções de Votação e Contagem de Preferências:
    vote(): Essa função verifica se o nome fornecido pelo eleitor corresponde a um candidato. Se sim, ela atualiza o array ranks com a preferência do candidato.
    record_preferences(): Com base nas preferências de um eleitor, essa função atualiza a matriz preferences com o número de eleitores que preferem um candidato sobre outro.
    add_pairs(): Esta função cria pares de candidatos onde um é preferido sobre o outro, baseando-se nas preferências registradas na matriz preferences.
  6. Função de Ordenação de Pares:
    sort_pairs(): Ordena os pares de candidatos em ordem decrescente com base na força da vitória, que é determinada pelo número de eleitores que preferem um candidato sobre o outro.
  7. Função de Bloqueio de Pares:
    lock_pairs(): Esta função bloqueia pares de candidatos no grafo de candidatos, garantindo que não ocorram ciclos. Ciclos ocorreriam se um candidato pudesse ser alcançado por meio de um caminho entre outros candidatos.
  8. Função de Impressão do Vencedor:
    print_winner(): Esta função determina e imprime o vencedor da eleição, que é o candidato sem arestas apontando para ele no grafo.
  9. Função de Verificação de Ciclos:
    check_for_cycles(): Uma função recursiva usada para verificar se a adição de uma aresta entre um par específico resultaria em um ciclo no grafo de candidatos. Isso evita a formação de ciclos durante o bloqueio de pares.

Em resumo, este código implementa o algoritmo de votação de Tideman, que é um sistema de votação preferencial usado para determinar um vencedor com base nas preferências dos eleitores. O código faz uso de matrizes para armazenar informações sobre os candidatos e os votos, e implementa várias funções para calcular as preferências, criar pares, classificá-los e bloqueá-los no grafo de candidatos. Por fim, ele determina e imprime o vencedor da eleição.

cadmusicas e guesssong

cadmusicas: Processamento de Arquivos de Áudio

Este trecho de código trata de processar arquivos de áudio no formato WAV. Ele executa as seguintes etapas:

  1. Lê os dados de áudio de um arquivo de entrada.
  2. Modifica o cabeçalho para incluir informações sobre a música e o artista.
  3. Escreve o áudio modificado juntamente com o novo cabeçalho em um arquivo de saída.

Os arquivos de cabeçalho necessários são incluídos:

  • stdint.h: Fornece tipos inteiros com larguras especificadas.
  • stdio.h: Fornece funções padrão de entrada/saída.
  • stdlib.h: Fornece funções de alocação de memória e de saída.
  • string.h: Fornece funções de manipulação de strings.
  • "arquivowav.h": Inclui um arquivo de cabeçalho definido pelo usuário chamado "arquivowav.h".

Definições e Constantes:

As constantes false e true são definidas com os valores 0 e 1, respectivamente.

Declarações de Funções:

São declarados os protótipos das funções check_format() e get_block_size().

Função main():

  1. Verifica os argumentos da linha de comando para garantir que sejam exatamente dois.
  2. Abre um arquivo de entrada para leitura.
  3. Lê o cabeçalho WAV do arquivo de entrada.
  4. Chama a função check_format() para verificar se o arquivo está no formato WAV.
  5. Abre um arquivo de saída para escrita.
  6. Obtém do usuário os nomes da música e do artista.
  7. Modifica o cabeçalho WAV com as informações fornecidas.
  8. Escreve o novo cabeçalho no arquivo de saída.
  9. Calcula o tamanho do bloco usando a função get_block_size().

guesssong.c: Jogo de Adivinhação de Músicas

Este trecho de código apresenta um jogo onde os jogadores ouvem trechos de músicas e tentam adivinhar as informações da música e do artista. Os pontos são concedidos por respostas corretas.

Os arquivos de cabeçalho necessários são incluídos da mesma forma que no primeiro trecho de código.

Constantes e Definições:

São definidas as constantes LOWER e UPPER, além da constante QUANT_SONGS que indica a quantidade de músicas no jogo.

Estruturas de Dados:

É criada uma estrutura chamada song que armazena informações sobre uma música, incluindo o cabeçalho WAV e amostras de áudio.

Declarações de Funções:

São declarados os protótipos das funções create_chosen_song(), calculate_points() e look_for_wav_files().

Função main():

  1. Solicita os nomes dos jogadores.
  2. Executa um loop onde três músicas são selecionadas aleatoriamente.
  3. Cria um arquivo de áudio para a música escolhida.
  4. Os jogadores tentam adivinhar os nomes das músicas e dos artistas.
  5. Os pontos são calculados com base nas respostas dos jogadores.
  6. O vencedor é determinado e exibido.

Função look_for_wav_files():

Procura por arquivos WAV no diretório e carrega as informações das músicas na estrutura de dados musicas.

Funções create_chosen_song() e calculate_points():

Essas funções são responsáveis por criar o arquivo de áudio selecionado e calcular os pontos, respectivamente.
